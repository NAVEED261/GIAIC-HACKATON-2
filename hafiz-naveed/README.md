## Phase-1: Console-Based Todo System

### Scope

**Phase-1 delivers**:
- Command-line Todo application written in Python
- In-memory task storage (no database)
- Menu-driven user interface
- Core operations: Add, List, Update, Delete, Mark Complete

**Phase-1 does NOT include**:
- Web interface, APIs, or database
- FastAPI, Next.js, or any web frameworks
- Authentication, multi-user support, or cloud deployment
- Persistence (tasks are lost on exit)
- AI or advanced features

### Quick Start

```bash
# Prerequisites: Python 3.8+

# Installation
pip install -r requirements.txt

# Run the application
python src/main.py

# Expected Output
===== Todo Menu =====
1. Add Task
2. List Tasks
3. Update Task
4. Delete Task
5. Mark Task Complete
6. Exit
Choose an option (1-6):
```

### Features

| Feature | Capability |
|---|---|
| **Add Task** | Create new task with title; auto-assigned ID |
| **List Tasks** | View all tasks with ID, title, and status |
| **Update Task** | Modify task title by ID |
| **Delete Task** | Remove task by ID |
| **Mark Complete** | Change task status from "Pending" to "Completed" |
| **Exit** | Gracefully terminate application |


## Agent Architecture

Phase-1 implements a **two-agent system**:

### HafizNaveed (Main/Orchestrator Agent)

**Responsibilities**:
- Display menu and handle user interaction
- Capture and validate user input
- Route requests to TodoActionAgent
- Display results and error messages
- Control application flow

**Location**: `src/cli.py`

**Responsibilities**:
- Manage in-memory task storage
- Execute all task operations (CRUD)

- Generate unique IDs
- Maintain state consistency
